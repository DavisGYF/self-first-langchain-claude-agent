# 阿里云OCR配置指南

## 📋 概述

本项目已集成阿里云OCR服务，相比Tesseract.js有以下优势：
- ✅ **无需下载语言包** - 首次使用即可识别
- ✅ **识别精度更高** - 阿里云专业OCR引擎
- ✅ **支持更多语言** - 覆盖全球主要语言
- ✅ **响应速度更快** - 云端高性能处理

## 🚀 快速开始

### 第一步：获取阿里云AccessKey

1. **登录阿里云控制台**
   - 访问 [阿里云官网](https://www.aliyun.com)
   - 使用你的账号登录

2. **创建AccessKey**
   - 进入 [AccessKey管理页面](https://ram.console.aliyun.com/manage/ak)
   - 点击「创建AccessKey」
   - 选择「继续使用AccessKey」
   - 保存生成的 `AccessKey ID` 和 `AccessKey Secret`

   ⚠️ **重要提示**：
   - AccessKey Secret只在创建时显示一次，请妥善保存
   - 建议定期更换AccessKey以保证安全

### 第二步：开通OCR服务

1. **进入OCR产品页面**
   - 访问 [OCR产品页面](https://www.aliyun.com/product/ocr)
   - 点击「立即开通」

2. **选择计费方式**
   - **按量付费**：适合小规模使用，0.01-0.1元/次
   - **资源包**：适合大规模使用，有折扣优惠
   - 新用户有免费试用额度

3. **等待服务激活**
   - 通常几分钟内即可激活完成

### 第三步：配置环境变量

在项目根目录创建或编辑 `.env.local` 文件：

```bash
# 阿里云OCR配置
ALIYUN_ACCESS_KEY_ID=你的AccessKeyID
ALIYUN_ACCESS_KEY_SECRET=你的AccessKeySecret
```

**示例**：
```bash
ALIYUN_ACCESS_KEY_ID=LTAI5tKbV1u2Z3j4m5n6o7p8
ALIYUN_ACCESS_KEY_SECRET=KbV1u2Z3j4m5n6o7p8q9r0s1t2u3v4w5x6y7z8
```

### 第四步：重启应用

```bash
# 重启开发服务器
npm run dev
```

## 🛠️ 故障排除

### 常见问题

#### 1. 配置完成后无法识别

**检查清单**：
- ✅ AccessKey是否正确配置
- ✅ OCR服务是否已开通
- ✅ 账户余额是否充足
- ✅ 环境变量是否生效（重启应用）

#### 2. 返回"签名错误"

**可能原因**：
- AccessKey Secret配置错误
- 时间不同步
- 特殊字符未正确转义

**解决方案**：
1. 重新生成AccessKey
2. 检查系统时间是否准确
3. 确保Secret中没有特殊字符

#### 3. 识别精度不如预期

**优化建议**：
- 确保图片清晰度（建议300dpi以上）
- 避免图片倾斜
- 选择正确的语言类型
- 对于复杂场景，使用高精度版

#### 4. 调用频率限制

**错误信息**：`Request frequency is too high`

**解决方案**：
- 降低调用频率
- 联系阿里云提升配额
- 使用缓存机制避免重复识别

查看浏览器开发者工具：
- **Network**：检查API调用状态
- **Console**：查看错误日志
- **Response**：分析返回数据格式

## 🔒 安全建议

### AccessKey安全

1. **不要提交到代码仓库**
   - 将 `.env.local` 添加到 `.gitignore`
   - 使用环境变量管理工具

2. **定期更换**
   - 建议每3-6个月更换一次AccessKey
   - 离职员工立即禁用相关AccessKey

3. **权限控制**
   - 为项目创建专用RAM用户
   - 仅授予OCR服务权限
   - 避免使用主账号AccessKey

### 数据安全

1. **图片传输**
   - 使用HTTPS加密传输
   - 避免传输敏感信息

2. **临时文件**
   - 系统自动清理临时文件
   - 不会永久存储用户图片

3. **日志记录**
   - 避免记录用户图片内容
   - 定期清理日志文件
